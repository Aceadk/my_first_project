## I18N-001 - Remove remaining hardcoded user-facing strings

Status: ⚠️ partially done — onboarding localized, broad backlog tracked

- 🔧 **Localized the onboarding progress + nav widgets.**
  [`onboarding_progress.dart`](../../lib/presentation/widgets/onboarding_progress.dart)
  hardcoded its step names (`'Welcome'`, `'Verify phone'`, …), `'Step X of Y'`,
  `'Next: …'`, `'Almost done'`, and `'Skip'`; `OnboardingNavButtons` defaulted to
  hardcoded `'Next'`/`'Previous'` (callers don't pass labels). All now resolve
  through `AppLocalizations` (new keys `onboardingProgressNextStep`,
  `onboardingProgressAlmostDone`, `onboardingProgressSkipStep`,
  `onboardingStep{Welcome,VerifyPhone,EnterCode,BasicInfo,VerifyId,ProfileSetup}`,
  `commonPrevious`; reusing `onboardingStep`, `commonNext`, `commonSkip`). Added
  to the template `app_en.arb`; other locales fall back to English until
  translated. This closes the gap flagged in the Onboarding UI audit.
- 🔧 **Localized 6 more critical-flow files** (reusing existing keys + 3 new:
  `authVerifyPasswordTitle`, `subscriptionPaywallTitle`,
  `subscriptionCancelSubscription`):
  `phone_protection_screen` and `change_email_screen` ("Verify Password" /
  "Verify"), `subscription_settings_screen` ("Manage subscription" / "Cancel
  subscription"), `paywall_screen` ("Get Premium" / ToS / Privacy — also gained
  l10n access), `chat_input_bar` ("Photo" / "Video"), and `privacy_settings_screen`
  ("Privacy Policy"). Verified by the existing privacy/subscription/toggle tests.
- 🔧 **Fully localized `safety_screen.dart`** — the safety center turned out to be
  an essentially unlocalized screen (~78 user-facing strings: snackbars, dialogs,
  section bodies, the safety-tips card, date-plan cards, status badges, the
  create-plan form, and validation), now entirely routed through ~78 new `app_en.arb`
  keys (reusing `commonCancel`/`commonSubmit`), including a `{count, plural}` for
  "Shared with N contact(s)" and `{name}`/`{date}` placeholders. A source re-scan
  confirms zero hardcoded user-facing strings remain in the file; covered by
  [`safety_l10n_keys_test.dart`](../../test/features/safety/safety_l10n_keys_test.dart)
  (plural, placeholders, and English fallback for untranslated locales).
- ⚠️ **Open / tracked.** Remaining hardcoded copy is in the `calls` screens
  (call history / video-call titles), a likely-legacy
  `lib/presentation/screens/home/settings_screen.dart`, and the long tail in
  `hardcoded_strings.txt`. The full legal screens (privacy/terms/community) are
  English long-form copy — candidate for an approved exception list. I18N-001
  remains **in progress** (now narrowed to calls + the long tail).

## I18N-002 - RTL, locale formatting, text expansion

Status: ✅ verified + tested

- **RTL-aware layouts.** Features/presentation use **0** non-directional
  `EdgeInsets.only(left:/right:)` versus 108 `EdgeInsetsDirectional` and 130
  `AlignmentDirectional` usages (only 2 non-directional `Alignment`), so padding
  and alignment mirror correctly under RTL. `GlobalWidgetsLocalizations` supplies
  directionality for `ar`/`ur`.
- **Locale-aware formatting.**
  [`date_time_formatter.dart`](../../lib/core/utils/date_time_formatter.dart) uses
  `intl` `DateFormat` (`.jm`/`.yMMMd`/`.Md`) with the active locale.
- **Text expansion.** The onboarding header was hardened against 2× scaling and
  long strings (ONBOARD-UI-003); a new RTL + 2×-scale case for the shared header
  is in
  [`onboarding_progress_text_scale_test.dart`](../../test/presentation/widgets/onboarding_progress_text_scale_test.dart).

## I18N-003 - Pluralization and embedded-text assets

Status: ✅ verified

- **Pluralization** uses proper ICU `{count, plural, =1{…} other{…}}` syntax —
  16 plural keys in the template (`timeMinutesAgo`, `a11yUnreadMessages`, …) —
  rather than naive string concatenation.
- ⚠️ One naive count string remains: `daily_likes_service.dart` builds
  `'Only $remaining likes left today!'` (hardcoded + non-plural) — folded into
  the I18N-001 backlog.
- **Embedded-text assets.** No localizable baked-in text found; the only
  text-bearing assets are the brand wordmark/app icon (`launch_wordmark.png`,
  `app_icon*`), which are brand marks and an acceptable exception.
